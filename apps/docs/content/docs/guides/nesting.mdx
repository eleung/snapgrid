# Nested grids

A grid can live **inside a tile of another grid**. Because each grid sits under its **own** dnd-kit
`DragDropProvider`, the inner and outer grids run independent drag sessions. Dragging an inner tile
rearranges only the inner grid, and dragging the outer tile carries the whole nested grid along.

<NestedDemo />

The one rule that keeps them from fighting: **the outer grid must drag from a handle that sits
outside the inner grid.** Give the outer tile a header (the drag handle) and put the inner grid in
the body. A pointer-down inside the inner grid then never arms the outer drag.

Each grid is its **own** `DragDropProvider`, and — as everywhere — `useGridContainer` runs in a child
host *inside* that provider:

```tsx
import { DragDropProvider } from "@dnd-kit/react";
import { GridDragOverlay, useGridContainer, useGridItem } from "@snapgridjs/react";

// The outer grid: its own provider wrapping the host.
function OuterGrid({ outer, width, setOuter }) {
  return (
    <DragDropProvider>
      <OuterBoard outer={outer} width={width} setOuter={setOuter} />
    </DragDropProvider>
  );
}

function OuterBoard({ outer, width, setOuter }) {
  const { containerProps, group } = useGridContainer({
    layout: outer,
    width,
    onLayoutChange: setOuter,
    dragConfig: { handle: ".panel-head" }, // handle sits OUTSIDE the inner grid
  });
  return (
    <>
      <div {...containerProps}>
        {outer.map((it) => (
          <Tile key={it.i} id={it.i} group={group}>
            {it.i === "panel" ? (
              <div className="panel">
                <div className="panel-head">Nested board</div> {/* outer drag handle */}
                <div className="panel-body">
                  <InnerGrid /> {/* its OWN provider — see below */}
                </div>
              </div>
            ) : (
              it.i
            )}
          </Tile>
        ))}
      </div>
      <GridDragOverlay>{({ item }) => (item ? <div className="tile">{item.i}</div> : null)}</GridDragOverlay>
    </>
  );
}

function Tile({ id, group, children }) {
  const { ref, style } = useGridItem(id, group);
  return (
    <div ref={ref} style={style} className="tile">
      {children}
    </div>
  );
}

// InnerGrid follows the exact same shape — its own <DragDropProvider> wrapping a
// host that calls useGridContainer + useContainerWidth for the inner board.
```

## Why it works

Each grid mounts its **own** dnd-kit `DragDropProvider`, so the inner grid's draggables, droppable
surface, and drag session are entirely separate from the outer's. The handle gating
(`dragConfig.handle`) is what keeps the shared pointer event from arming both. The inner tiles sit
outside the outer's handle, so only the inner grid responds.

<Callout type="warning">
  Without a handle the outer tile drags from anywhere, including the inner grid, so a single
  pointer-down would arm **both** grids. Always give the outer grid a handle that lives outside the
  inner area.
</Callout>

## Limits

- **Size the inner grid to its cell.** The inner grid measures its own container with
  `useContainerWidth`, so it reflows naturally as the outer tile is resized.

<Callout type="warning">
  **Nested grids are isolated.** Each needs its **own** `DragDropProvider` — share one and the inner
  grid is ambiguous. Tiles can't cross between levels, either; to move tiles between grids, make them
  **siblings** (not nested) and use [cross-grid dragging](/docs/guides/cross-grid).
</Callout>
