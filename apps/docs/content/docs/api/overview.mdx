# API Overview

A map of the public surface. Follow a link for details.

## `@snapgridjs/react`

The main package. Hooks (the headless layer), turnkey components, and a re-export of the engine
types. Headless grids render inside a dnd-kit `DragDropProvider`; `<GridLayout>` bundles one.

### Hooks (headless)

| Export | Returns |
| --- | --- |
| [`useGridContainer`](/docs/api/hooks#usegridcontainer) | `{ containerProps, group, isDropTarget, controller }` — the grid host |
| [`useGridItem`](/docs/api/hooks#usegriditem) | `{ ref, handleRef, style, isDragging, item }` |
| [`useGridResizeHandle`](/docs/api/hooks#usegridresizehandle) | `{ ref, handleProps, isResizing }` |
| [`useGridPlaceholder`](/docs/api/hooks#usegridplaceholder) | `{ item, style } \| null` |
| [`useContainerWidth`](/docs/api/hooks#usecontainerwidth) | `{ width, mounted, containerRef }` |
| [`useResponsiveLayout`](/docs/api/hooks#useresponsivelayout) | `{ breakpoint, cols, layout, onLayoutChange }` |

### Components (turnkey)

| Export | Purpose |
| --- | --- |
| [`GridLayout`](/docs/api/components#gridlayout) | Drop-in controlled grid with keyed children (bundles the provider). |
| [`ResponsiveGridLayout`](/docs/api/components#responsivegridlayout) | Per-breakpoint layouts. |
| [`GridItem`](/docs/api/components#griditem) | A positioned tile with default markup + handles. |
| [`GridPlaceholder`](/docs/api/components#gridplaceholder) | Default landing-cell marker. |
| [`SnapGridGroup`](/docs/api/components#snapgridgroup) | Shares one provider across sibling grids (cross-grid). |

### Re-exported dnd-kit primitives

For convenience (one import source), `@snapgridjs/react` re-exports the dnd-kit pieces you commonly
reach for with a grid: `DragOverlay` (the floating drag preview), `useDraggable` / `useDroppable`
(e.g. an external-drop palette), and `Feedback` / `PointerSensor` / `KeyboardSensor`. `DragDropProvider`
is *not* re-exported — import it from `@dnd-kit/react`.

For the floating preview, prefer **`GridDragOverlay`** — snapgrid's grid-aware wrapper over dnd-kit's
`DragOverlay`. It resolves the dragged item (render-prop receives `{ item, source }`) and bakes in the
out-of-flow style so the overlay can't flash full-width at the bottom of the grid before dnd-kit
positions it (see [Headless usage](/docs/guides/headless#dragoverlay)). The raw `DragOverlay` and the
bare `dragOverlayStyle` are exported too, for full control; `<GridLayout>` applies the style for you.

### Utilities & compactors

`DEFAULT_BREAKPOINTS`, `DEFAULT_BREAKPOINT_COLS`, and the built-in compactors
`verticalCompactor`, `horizontalCompactor`, `noCompactor`, `getCompactor`.

### Types & config

`Layout`, `LayoutItem`, `GridConfig`, `DragConfig`, `ResizeConfig`, `DropConfig`, `GridDropData`,
`GridEventCallback`, `Compactor`, `CompactType`, `Breakpoints`, `BreakpointCols`,
`ResponsiveLayouts`, `PositionParams`, `ResizeHandleAxis`. See [Types & config](/docs/api/types).

## `@snapgridjs/core`

The framework-agnostic engine: geometry, move/resize/insert/remove operations, and the drag
session state machine. Most apps never import it directly. See
[Core engine & extras](/docs/api/core).

## `@snapgridjs/extras`

Optional packers: `masonryCompactor`, `gravityCompactor`, `shelfCompactor`, plus `wrapCompactor`
and the fast O(n log n) compactors re-exported from react-grid-layout. See
[Core engine & extras](/docs/api/core#snapgridjsextras).
