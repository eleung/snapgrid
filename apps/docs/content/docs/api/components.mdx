# Components

The turnkey layer — a thin, react-grid-layout-style shell over the [hooks](/docs/api/hooks).
`<GridLayout>` bundles the dnd-kit `DragDropProvider`, the container, the items, the placeholder, and
the [`DragOverlay`](https://next.dndkit.com), so you never touch dnd-kit directly. For full control
of the markup, or to compose with other dnd-kit draggables, use the hooks.

## GridLayout

The drop-in controlled grid. Children are keyed by their layout item's `i`. It supplies its own
`DragDropProvider`; nest several (or wrap them in [`SnapGridGroup`](#snapgridgroup)) and they share
one.

```tsx
<GridLayout
  layout={layout}
  width={width}
  onLayoutChange={setLayout}
  gridConfig={{ cols: 12, rowHeight: 60, margin: [12, 12] }}
  resizeConfig={{ handles: ["se", "e", "s"] }}
>
  {layout.map((item) => (
    <div key={item.i}>{item.i}</div>
  ))}
</GridLayout>
```

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `layout` | `Layout` | — | **Required.** Controlled array of items. Never mutated. |
| `width` | `number` | — | **Required.** Container width in px (from `useContainerWidth`). |
| `onLayoutChange` | `(layout: Layout) => void` | — | Called with the next layout when an interaction commits. |
| `gridConfig` | `Partial<GridConfig>` | see [GridConfig](/docs/api/types#gridconfig) | Columns, row height, margins, padding, maxRows. |
| `dragConfig` | `DragConfig` | — | [Drag behaviour](/docs/guides/dragging). |
| `resizeConfig` | `ResizeConfig` | `{ handles: ["se"] }` | [Resize behaviour](/docs/guides/resizing). |
| `dropConfig` | `DropConfig` | `{ enabled: false }` | [Accept external draggables](/docs/guides/external-drop). |
| `compactor` | `Compactor` | `verticalCompactor` | [Packing algorithm](/docs/guides/compaction). |
| `isDraggable` | `boolean` | `true` | Grid-level drag toggle. |
| `isResizable` | `boolean` | `true` | Grid-level resize toggle. |
| `autoSize` | `boolean` | `true` | Grow the container height to fit content. |
| `id` | `string` | auto (`useId`) | Stable id for the droppable surface. |
| `onDragStart` `onDrag` `onDragStop` | `GridEventCallback` | — | [Drag lifecycle](/docs/guides/events). |
| `onResizeStart` `onResize` `onResizeStop` | `GridEventCallback` | — | Resize lifecycle. |
| `onDrop` | `(layout, item, event) => void` | — | Fired when an external draggable is dropped in. |
| `className` | `string` | — | Appended to `snapgrid`. |
| `style` | `CSSProperties` | — | Merged over the surface's positioning style. |

## ResponsiveGridLayout

Switches columns and layout by breakpoint as `width` changes. See
[Responsive layouts](/docs/guides/responsive).

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `width` | `number` | — | **Required.** Container width in px. |
| `layouts` | `ResponsiveLayouts` | — | **Required.** Per-breakpoint layout map. |
| `onLayoutChange` | `(layout, layouts) => void` | — | Active layout **and** the updated map. |
| `onBreakpointChange` | `(breakpoint, cols) => void` | — | Fired when the active breakpoint changes. |
| `breakpoints` | `Breakpoints` | `DEFAULT_BREAKPOINTS` | Breakpoint → min width (px). |
| `cols` | `BreakpointCols` | `DEFAULT_BREAKPOINT_COLS` | Breakpoint → column count. |
| `rowHeight` | `number` | `150` | Row height in px. |
| `margin` | `[number, number]` | `[10, 10]` | Gap between items. |
| `containerPadding` | `[number, number] \| null` | `null` | Surface padding; falls back to `margin`. |
| `compactor` `dragConfig` `resizeConfig` `isDraggable` `isResizable` `autoSize` `className` `style` | | | As on `GridLayout`. |

## GridItem

A positioned tile with stable classes (`snapgrid-item`, `data-grid-id`, `data-dragging`) and the
configured resize handles. Memoized, so re-rendering the surface doesn't re-render every tile. Use
it inside your own surface, or let `GridLayout` create them.

| Prop | Type | Description |
| --- | --- | --- |
| `id` | `string` | **Required.** Matches the layout item's `i`. |
| `group` | `string` | **Required.** The owning grid's id (from its `useGridContainer`). |
| `children` | `ReactNode` | Tile content. |
| `className` | `string` | Appended to `snapgrid-item`. |
| `style` | `CSSProperties` | Merged over the positioning style. |

## GridPlaceholder

Renders the landing-cell marker, or nothing when idle. Takes the grid's `group`. Customize with
`className` / `style`, or use [`useGridPlaceholder`](/docs/api/hooks#usegridplaceholder) for full
control.

| Prop | Type | Description |
| --- | --- | --- |
| `group` | `string` | **Required.** The owning grid's id. |
| `className` | `string` | Appended to `snapgrid-placeholder`. |
| `style` | `CSSProperties` | Merged over the default look. |

<Callout type="info">
  There is no snapgrid drag-overlay component. The floating preview is dnd-kit's
  [`DragOverlay`](https://next.dndkit.com) — `GridLayout` renders one for you; in the headless layer
  you render it yourself. See [Headless usage](/docs/guides/headless).
</Callout>

## SnapGridGroup

Shares one dnd-kit `DragDropProvider` across several sibling grids so tiles can be dragged between
them. Takes only `children` — a thin wrapper over `DragDropProvider` (nesting `GridLayout`s already
shares a provider, so this is just explicit). See [Cross-grid dragging](/docs/guides/cross-grid).

<Callout type="warning">
  Item ids must be unique across all grids that share a provider.
</Callout>
